## 对应需求（handoff）

`PROJECT_HANDOFF.md` §4 Current Work Node — **v0.01.0029 — 9-key whole-word candidates + syllable fallback**：

1. **Case A**：完整数字串有多音节整词时，候选栏优先整词；一点上屏。
2. **Case B**：点左侧拼音后进入逐音节；剩余数字继续。
3. **Case C**：无整词（或需自造词）时逐音节组词，提交后可整词召回。
4. **回归**：组字中 Backspace/DEL 不崩溃。

## 问题与建议

1. **P0（体验/功能）**：`726` 应在左侧同时列出 `pan/san/ran`（或与候选一致的默认音节）；当前「高亮 san、候选却像 pan」会误导，且阻断 `分盘` 类自造词。
2. **文档**：handoff Case C 写「`336726` 无整词」不准确 — 内置有 `fenran=愤然…`。应改为「即使有其它整词命中，仍可通过点 `fen` 走音节路径」。
3. **自动化**：IME 按键不在 uiautomator 树内；EditText 的 `text` 属性常缺失，需以截图为准。左侧拼音命中对 Y 敏感。
4. **Push 建议**：**Case A/B 可接受则源码可进主干**；若必须验收「分盘学习召回」，应先修 `726` 音节列表再测 Case C 全闭环。

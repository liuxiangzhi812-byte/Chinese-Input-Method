# v0.02.0003 - 2026-07-13

执行者: Grok (CLI Agent)

## 背景

v0.02.0002 开发快照在真机上可复现运行时词库索引 OOM（堆上限约 402MB），学习/导入时全量重建前缀索引导致进程退出。本版本优先修复该 P0，并交付 P1 中的管理服务空闲关闭与完整候选下拉。

## 新增 / 修复

### P0 — 运行时词库索引 OOM 稳定性

- 拼音前缀索引改为 `SortedPrefixIndex`：单一有序 key 数组 + 二分区间查找，不再为每个前缀维护 `HashMap` 桶复制完整拼音列表。
- 数字 / 数字前缀索引改为 `SortedDigitIndex`：按数字串排序的并行数组，同时支持 exact 与 prefix。
- `addUserCandidate` / `mergeUserCandidateIntoRuntime` **不再**对整库调用 `applyDictionaryLayers`：
  - 已有拼音：仅浅拷贝候选 map 并更新该 key 的候选数组，复用既有索引。
  - 新拼音：对数字索引与前缀索引做单 key 增量插入。
- 全量导入/重载仍走原子 `applyDictionaryLayers`，但基于紧凑索引，峰值内存显著低于旧实现。
- 单拼候选上限由 20 提高到 **512**，避免整表截断导致候选下拉永远看不到后半字。
- 增加轻量诊断日志：`PinyinDict` 记录 rebuild reason、durationMs、keys、heapBefore/heapAfter（不记录词条内容）。

### P1 — 电脑管理 10 分钟空闲关闭

- `ComputerDictionaryService` 在开启后启动 10 分钟空闲计时。
- **会重置计时**：连接批准、通过鉴权的 status/import/export/clear/disconnect。
- **不会重置计时**：UDP 发现、未授权/错误会话请求。
- 超时后关闭 TCP/UDP、使会话失效、移除前台通知，服务回到未开启状态。

### P1 — 完整可滚动候选下拉

- 展开面板高度调整为 192dp，显式垂直滚动条。
- 展开面板始终填充**完整**当前组字候选数组（不复用紧凑行分页截断）。
- 配合候选上限提高，单拼如 `ji` 可在下拉中滚动到完整结果集（含「激」），不设单词特判。

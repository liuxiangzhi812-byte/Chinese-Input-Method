# v0.02.0002 - 2026-07-11

执行者: Codex

## 新增

- 手机设置页新增“电脑词库管理”。服务默认关闭，仅在用户主动开启后监听同一局域网。
- 电脑端新增本地网页管理程序，网页只开放在 `127.0.0.1:37620`，支持搜索并选择已开启服务的手机。
- Windows 端以完整便携版 ZIP 交付；必须整体解压并保留 EXE 旁的 `app`、`runtime` 文件夹，避免单独复制 EXE 导致 JVM 无法启动。
- Windows 便携版目录、EXE 和启动配置统一改为英文名称，规避 Explorer 双击中文 `jpackage` 启动器时出现 `Failed to launch JVM` 的环境兼容问题；网页内容保持中文。
- 连接请求必须在手机通知中确认；确认后手机签发临时随机会话令牌，服务关闭或重新连接即失效。
- 管理命令使用递增序号，拒绝重复和旧请求，降低网页刷新和网络重发造成重复操作的风险。
- 电脑网页可查看人工词条、自学习词条与累计选择数量。
- 支持电脑端预览 UTF-8 TSV、导入并替换人工词库、导出仅人工词库、导出人工与自学习合并词库。
- 清空人工词库和清空学习数据必须分别在手机通知中再次确认。

## 通信边界

- 当前版本只支持电脑与手机处于同一局域网，不依赖云服务、外部服务器、VPN、数据线或 ADB。
- UDP `37622` 用于发现手机，手机 TCP `37621` 用于词库操作；传输内容为明文，仅建议在可信家庭网络使用。
- 电脑助手拒绝选择公网地址；手机服务通过持续通知显示运行状态，并可随时关闭。
- 电脑助手校验浏览器的本机 Host 与 Origin，拒绝其他网站借浏览器向本地管理接口代发请求。
- 手机端继续执行最终 TSV 校验、10 MB 大小限制、原子替换与后台索引重建，电脑预览不取代手机校验。

## 已记录但本版本不修改

- 用户发现输入单拼 `ji` 时，候选界面无法找到“激”，但完整拼音仍能命中“激动”，说明词库已有该字而单拼候选提取或显示范围不完整。
- 后续版本将统一实现可下拉、可上下滑动的完整字词候选面板，并检查候选截断链路；不得只为 `ji` 或“激”增加特殊词库规则。

## 真机问题修复

- 在 OnePlus 7 Pro 真机复现“电脑请求手机确认后 App 闪退”：电脑网页轮询请求状态时，Android 12 缺少 `URLDecoder.decode(String, Charset)`，后台线程触发 `NoSuchMethodError` 并导致整个进程退出。
- 查询参数解析改用 Android 12 可用的 UTF-8 解码重载。协议与页面流程不变；需重新构建、安装并复测连接确认和后续轮询。

## 工程复核（2026-07-13）

- 用户反馈正常打字仍频繁、无明显规律地闪退。当前设备 logcat 已确认多次 `OutOfMemoryError`：`PinyinDictionary.buildPinyinPrefixIndex` 在自造词写入或词库重载后重建完整运行时索引，峰值达到 Android 12 的约 402MB 堆上限。
- 该问题定为 P0 发布阻塞项。后续必须改成紧凑前缀查询结构，并避免每个自造词触发全量索引重建；不以 `largeHeap`、捕获 OOM 或删减词库代替结构性修复。
- 电脑管理服务后续增加 10 分钟无有效操作自动关闭：关闭发现和 TCP 监听、销毁令牌、移除前台通知并回到未开启状态。未授权请求和局域网发现包不延长计时。
- 对测试报告“便携包缺 `runtime\\bin\\java.exe` 即无法运行”的判据提出异议：`jpackage` EXE 可直接加载私有 `jvm.dll`，且英文启动器已在端口空闲条件下实测启动并返回 HTTP 200。应以实际双击结果和启动日志判断，不应要求私有运行时必须提供 `java.exe`。

## Known JDK / JBR Issue

The Codex environment previously found these JetBrains runtimes:

```text
C:\Program Files\JetBrains\PyCharm 2025.1.1.1\jbr
C:\Program Files\JetBrains\PhpStorm 2025.1.1\jbr
```

They include `javac.exe`, but no `jlink.exe`.

Android Gradle builds for this project currently need a full JDK/JBR with `jlink.exe` because AGP transforms:

```text
C:\Users\76556\AppData\Local\Android\Sdk\platforms\android-36.1\core-for-system-modules.jar
```

If Gradle fails with:

```text
jlink executable ...\bin\jlink.exe does not exist
```

the source code is not necessarily broken. Use Android Studio's embedded runtime if available, or install/configure a full JDK 17/21 that includes `jlink.exe`, then set `JAVA_HOME` to that JDK before running Gradle.
